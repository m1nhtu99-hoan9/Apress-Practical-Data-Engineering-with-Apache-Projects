# Dev Log

Date: 2026-03-05
Scope: Kotlin Jupyter notebooks in `chapter-02` (`spark-iceberg` image)

## Problem

- Kotlin notebook cells failed on:
  - `Unresolved reference: apache`
  - `Unresolved reference: SparkSession`
- This occurred in the first Spark import cell of the "Getting Started" notebooks.

## Root Cause

1. Kotlin kernel classpath did not include Spark/Iceberg jars from `/opt/spark/jars`.
2. Spark on Java 17 inside Kotlin kernel needs module opens (`--add-opens`) that `spark-submit` normally handles.
3. Existing notebook cells relied on Spark defaults/catalog config being present, but Kotlin kernel startup is not the same path as `pyspark`.

## Changes

File changed:
- `spark/Dockerfile`

What was added:
1. Build-time patch step that:
  - Symlinks all jars from `/opt/spark/jars` into Kotlin kernel jars directory.
  - Appends jar names to Kotlin kernel `jar_args.json` classpath.
2. Runtime env for Kotlin kernel:
  - `SPARK_CONF_DIR=/opt/spark/conf`
  - `KOTLIN_JUPYTER_KERNEL_EXTRA_JVM_OPTS` including:
    - Required Java 17 `--add-opens` flags.
    - `-Dio.netty.tryReflectionSetAccessible=true`.
    - `-Dspark.master=local[*]` so `SparkSession.builder().getOrCreate()` works in Kotlin notebooks without extra boilerplate.
    - Spark/Iceberg catalog defaults (`spark.sql.catalog.demo.*`, `spark.sql.defaultCatalog`, etc.) aligned with `spark-defaults.conf`.

## Why this approach

- Makes the fix persistent at image level (no per-notebook bootstrapping required).
- Avoids dependency/version drift from ad-hoc `@file:DependsOn(...)` cells.
- Keeps Kotlin notebook behavior consistent with the intended Iceberg REST + MinIO setup.

## Validation Plan

After rebuilding image:

1. Rebuild and restart:
   - `docker compose build spark-iceberg`
   - `docker compose up -d spark-iceberg`
2. In Kotlin kernel notebook, run:
   - `import org.apache.spark.sql.SparkSession`
   - `val spark = SparkSession.builder().appName("Jupyter").getOrCreate()`
   - `spark.version()`
3. Confirm catalog defaults:
   - `spark.conf().get("spark.sql.defaultCatalog")` -> `demo`
   - `spark.conf().get("spark.sql.catalog.demo.uri")` -> `http://rest:8181`
