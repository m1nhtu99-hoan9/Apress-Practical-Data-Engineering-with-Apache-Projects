# Dev Log

Date: 2026-03-01
Scope: `chapter-02` Docker Compose + Spark image/data initialization flow

## Findings

1. Build frontend pull failed before image build started.
- Symptom: `failed to resolve source metadata for docker.io/docker/dockerfile:1` with TLS handshake timeout.
- Root cause: `# syntax=docker/dockerfile:1` triggered a pull from Docker Hub for the Dockerfile frontend.
- Fix: Removed the syntax directive from `spark/Dockerfile`.

2. Compose build raced on a shared image tag.
- Symptom: `image "chapter-02-spark-iceberg:latest": already exists` during `docker compose build`.
- Root cause: both `spark-iceberg` and `data-init` had `build: spark/` and exported the same tag in parallel.
- Fix: Kept `build` only on `spark-iceberg`; `data-init` now reuses `image: chapter-02-spark-iceberg:latest`.

3. `data-init` exited with code `8`.
- Symptom: `service "data-init" didn't complete successfully: exit 8`.
- Root cause: `wget` to NYC Socrata URL for `film_permits.json` returned HTTP `403 Forbidden`.
- Fix: Kept the requested `wget` command/flags but made that download non-fatal, logging a warning and continuing.

4. Failed download left an invalid zero-byte `film_permits.json`.
- Symptom: later runs skipped `film_permits.json` because file existed, even though it was empty.
- Root cause: existence check used `-f` only, and failed downloads could leave placeholder files.
- Fix:
  - Skip only when file is non-empty (`-s`).
  - Remove empty target files before retry.
  - Download to `*.part` temp files and `mv` on success (atomic finalization).
  - On film permit download failure, clean up `.part` and target placeholders.

## Current State

- `docker compose build` succeeds.
- `docker compose up -d` succeeds with:
  - `data-init` exiting `0` (required by `depends_on: condition: service_completed_successfully`).
  - `spark-iceberg`, `minio`, `mc`, and `rest` running as expected.
- If remote film permit download stays blocked (403), provide `spark/data/film_permits.json` locally to bypass network fetch.
